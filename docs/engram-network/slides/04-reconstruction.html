Memory Reconstruction

Nova doesn't retrieve memories — it relives them

The Neuroscience Insight
When you remember your last birthday, you don't replay a recording. Your brain reconstructs the experience from fragments — who was there, the emotion, the cake, the surprise. Each time you recall it, the reconstruction is slightly different, colored by your current state. This is why memories evolve. This is why the same event means different things at different times.
❌ Retrieval (what every AI does)
STORED:
"On March 5, the user deployed Nova to their home server using Docker Compose and switched from Cloudflare tunnels to Tailscale."
RETURNED:
"On March 5, the user deployed Nova to their home server using Docker Compose and switched from Cloudflare tunnels to Tailscale."
Verbatim. Static. Same text regardless of context.
✅ Reconstruction (Engram Network)
STORED (fragments):
entity:home-server fact:Tailscale episode:deployed pref:simpler fact:no-CF
CONTEXT: "debugging network issues"
"We moved to Tailscale after Cloudflare tunnels were causing issues. The home server is on the tailnet now — that's probably where to check connectivity."
CONTEXT: "planning infrastructure changes"
"Last time we simplified the stack by replacing Cloudflare tunnels with Tailscale on the home server — you preferred the simpler setup. Should we apply the same philosophy here?"
Same fragments → different reconstruction based on what matters NOW.

Reconstruction Process
A lightweight prompt assembles fragments into a coherent memory narrative. Not an LLM call per recall — this is a template-driven assembly with context-aware ordering and emphasis.
For complex/nuanced memories:
Optional LLM reconstruction (Haiku-class) for natural narrative. Batched — multiple memories in one call.

🧠 Why This Makes Nova Feel Alive
Memories Evolve
Same event, reconstructed months later with new context, yields richer understanding. Like a person reflecting on past experiences.
First-Person Perspective
Nova's self-model colors the reconstruction. "I suggested Tailscale and you agreed — it was the right call" not "user switched to Tailscale."
Context Sensitivity
Debugging? The memory emphasizes technical details. Planning? It emphasizes the decision-making pattern. Same fragments, different focus.
